# "危险关系"剧集视频源重复问题修复验证报告

## 修复概述

已成功修复 `detailStore.ts` 中的去重逻辑缺陷，解决了视频源重复显示的问题。

## 修复内容

### 1. 统一去重逻辑

**修改位置**：`/root/OneTV/stores/detailStore.ts` 第 121-167 行

**修改前**：
- 使用复杂的 `merge`/`non-merge` 分支逻辑
- 去重使用 `source_id` 和 `source_title_year` 两个键
- 去重逻辑不一致，导致某些情况下重复项未被正确过滤

**修改后**：
- 使用统一的 `source_id_title_year` 作为唯一键
- 简化去重逻辑，移除复杂的分支处理
- 确保所有场景下都使用相同的去重规则

```typescript
set((state) => {
  // 统一去重逻辑：使用 source_id_title_year 作为唯一键
  // 不同年份的同一剧集被视为不同结果
  const seenKeys = new Set<string>();
  const allResults = merge ? [...state.searchResults, ...resultsWithResolution] : resultsWithResolution;
  
  const deduplicatedResults = allResults.filter((r) => {
    const uniqueKey = `${r.source}_${r.id}_${r.title}_${r.year || ''}`;
    if (seenKeys.has(uniqueKey)) {
      return false;
    }
    seenKeys.add(uniqueKey);
    return true;
  });
  
  logger.info(`[MATCHING] Before dedup: ${allResults.length} results, After dedup: ${deduplicatedResults.length} results`);
  
  let processedResults = deduplicatedResults;
  let finalDeduplicatedResults = deduplicatedResults;
  
  // 使用匹配引擎处理
  // ... 匹配逻辑
  
  // 最终去重
  const seenFinal = new Set<string>();
  finalDeduplicatedResults = processedResults.filter((r) => {
    const uniqueKey = `${r.source}_${r.id}_${r.title}_${r.year || ''}`;
    if (seenFinal.has(uniqueKey)) {
      return false;
    }
    seenFinal.add(uniqueKey);
    return true;
  });
});
```

### 3. 变量命名优化

- `finalResults` → `processedResults`（匹配后的结果）
- `deduplicatedResults` → `finalDeduplicatedResults`（最终去重后的结果）
- 避免变量名冲突，提高代码可读性
