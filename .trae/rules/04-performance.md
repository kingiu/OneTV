# 安全规范（必须遵守）

## 输入验证与注入防护
- 所有用户输入（URL 参数、请求体、表单、文件上传）必须经过验证和清理。
- SQL 查询：使用 ORM 参数化或查询构建器，禁止字符串拼接。
- Shell 命令：使用 `child_process.execFile` 而非 `exec`，避免注入。
- HTML 输出：在浏览器端使用 `textContent` 而非 `innerHTML`；服务端模板必须转义。

## 认证与授权
- 所有非公开路由必须检查认证（JWT / Session）。
- 权限检查应使用中间件或装饰器，不在业务逻辑中分散。
- 禁止在客户端存储敏感信息（如 token 存在 `localStorage`？需评估风险并加密）。

## 密钥管理
- 所有密钥（API Key、数据库密码）必须通过环境变量注入，禁止硬编码。
- 示例配置文件（`.env.example`）必须提交，真实 `.env` 加入 `.gitignore`。

## 依赖安全
- 定期运行 `npm audit` / `yarn audit`，高危漏洞必须在 7 天内修复。
- 禁止引入未经验证的开源包（尤其是自动下载的二进制文件）。
