# 测试规范

## 单元测试
- 框架：Vitest（前端）/ Jest（Node.js）。
- 每个业务模块至少包含一个单元测试文件，测试核心逻辑。
- 测试描述使用中文：`it('当输入为负数时，应该返回 0')`。
- 覆盖率要求：行覆盖率 ≥80%，分支覆盖率 ≥70%。

## 集成测试
- 每个 API 端点至少有一个成功/失败场景的集成测试。
- 测试数据库使用独立实例（如 SQLite in-memory 或 Docker 临时库）。

## E2E 测试（前端项目）
- 关键用户流程（登录、下单）必须有 Playwright/Cypress 测试。
- 在 PR 合并前自动运行。

## 测试替身
- Mock 外部服务（HTTP、数据库）使用 `vi.mock` 或 `jest.mock`。
- 避免过度 Mock：只 Mock 直接依赖，不 Mock 间接依赖。
