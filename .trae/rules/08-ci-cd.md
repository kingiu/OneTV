# CI 流水线检查项（必须通过才能合并）

1. **代码检查**：ESLint (无 error) + Prettier (格式一致)
2. **类型检查**：`tsc --noEmit`
3. **单元测试**：全部通过，覆盖率达标
4. **集成测试**（如有）：API 测试通过
5. **安全扫描**：`npm audit` 无高危漏洞
6. **构建验证**：生产构建成功（`npm run build`）

## 自动化规则
- 禁止在 CI 中跳过任何检查（如 `--no-verify` 仅限本地临时使用）。
- 主分支（main/master）必须通过 PR 合并，且 PR 必须通过上述所有检查。
